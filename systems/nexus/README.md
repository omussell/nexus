## NEXUS

Provenance and relationships

When the data is processed, the processing itself creates metadata which is important. For example, if Vulcanus updates the metadata based on data from a ROR snapshot then it would be good to have a record of which ROR snapshot was used. This is described as provenance.

We also want to generate "relationships" metadata. For example, if we Vulcanus matching identifies that a ROR organization was the funder for a journal article, then we want to add metadata to that ROR organization to say: "this organization funded this article" and also add metadata to the journal article to say: "this journal article was funded by this organization".

Stored and provided as files that are in RDF format.
